# پیکربندی - تنظیمات ضروری

پیکربندی نحوه سفارشی کردن Open Notebook برای تنظیمات خاص خود است. این بخش آنچه را که باید بدانید را پوشش می دهد.

---

## چه چیزی به پیکربندی نیاز دارد؟

سه چیز:

1. **ارائه‌دهنده هوش مصنوعی** - از کدام سرویس LLM/Embedding استفاده می‌کنید (OpenAI، Anthropic، Ollama، و غیره)
2. **پایگاه داده** — نحوه اتصال به SurrealDB (معمولاً از پیش پیکربندی شده)
3. **سرور** - URL API، پورت ها، وقفه های زمانی (معمولاً به صورت خودکار شناسایی می شوند)

## فایل پیکربندی

بسته به تنظیمات خود از فایل مناسب استفاده کنید.

### «.env» (توسعه محلی)

فقط در صورتی از .env استفاده خواهید کرد که Open Notebook را به صورت محلی اجرا کنید.

```
واقع در: ریشه پروژه
استفاده برای: توسعه در دستگاه شما
قالب: KEY = مقدار، یک در هر خط
```

### «docker.env» (استقرار داکر)

اگر از docker-compose استفاده می‌کنید و ترجیح می‌دهید متغیرها را مستقیماً در فایل نوشتن قرار ندهید، از این فایل برای نگهداری متغیرهای محیط خود استفاده خواهید کرد. 
```
واقع در: ریشه پروژه (یا ./docker)
استفاده برای: استقرار Docker
قالب: مانند env
بارگیری شده توسط: docker-compose.yml
```

## مهمترین تنظیمات

همه تنظیمات ارائه شده در زیر باید در فایل محیطی شما قرار داده شوند (بسته به تنظیمات شما. env یا docker.env).


### پایگاه داده سورئال

این پایگاه داده ای است که توسط برنامه استفاده می شود.

```
SURREAL_URL=ws://surrealdb:8000/rpc
SURREAL_USER=ریشه
SURREAL_PASSWORD=root # تغییر در تولید!
SURREAL_NAMESPACE=open_notebook
SURREAL_DATABASE=نوت بوک باز
```

> تنها چیزی که ضروری است از دست ندهید، نام میزبان در «SURREAL_URL» است. بررسی کنید از چه URL بر اساس استقرار خود استفاده کنید، [اینجا] (database.md).


### ارائه دهنده هوش مصنوعی (مدارک اعتباری)

برای اینکه برنامه کار کند به LLM نیاز داریم. اعتبارنامه های ارائه دهنده هوش مصنوعی از طریق **تنظیمات UI** پیکربندی می شوند:

1. «OPEN_NOTEBOOK_ENCRYPTION_KEY» را در محیط خود تنظیم کنید (برای ذخیره اطلاعات کاربری لازم است)
2. خدمات را شروع کنید3. به **تنظیمات → کلیدهای API → افزودن اعتبار ** بروید
4. ارائه دهنده خود را انتخاب کنید، کلید API خود را جایگذاری کنید
5. **تست اتصال** → **کشف مدل******ثبت نام مدل**

```
# مورد نیاز در env. یا docker-compose.yml شما:
OPEN_NOTEBOOK_ENCRYPTION_KEY=کلید مخفی من
```

> **کاربران Ollama**: یک اعتبارنامه Ollama را در تنظیمات → کلیدهای API با URL پایه صحیح اضافه کنید. برای راهنمایی پیکربندی شبکه، [Ollama Setup] (ollama.md) را ببینید.

> **LM Studio / OpenAI-Compatible **: یک اعتبارنامه سازگار با OpenAI را در تنظیمات → کلیدهای API اضافه کنید. به [OpenAI-Compatible Guide] (openai-compatible.md) مراجعه کنید.


### URL API (اگر پشت پراکسی معکوس باشد)
فقط در صورت استقرار در پروکسی یا تغییر اطلاعات پورت باید نگران این موضوع باشید. در غیر این صورت، از این کار بگذرید.

```
API_URL=https://your-domain.com
# معمولاً به صورت خودکار شناسایی می شود. فقط در صورت نیاز تنظیم کنید.
```

تشخیص خودکار برای اکثر تنظیمات کار می کند.

## پیکربندی بر اساس سناریو

`` env

```

`` env

# پروکسی معکوس HTTPS را مدیریت می کند
```

### سناریوی 4: استفاده از Ollama به صورت محلی
`` env
# در env:
OPEN_NOTEBOOK_ENCRYPTION_KEY=کلید مخفی من
# سپس اعتبار Ollama را در تنظیمات → کلیدهای API اضافه کنید
```

### سناریوی 5: استفاده از Azure OpenAI
`` env
# در docker.env:
OPEN_NOTEBOOK_ENCRYPTION_KEY=کلید مخفی من
# سپس اعتبار Azure OpenAI را در تنظیمات → کلیدهای API اضافه کنید

## چگونه پیکربندی را اضافه کنیم

### روش 1: تنظیمات UI (برای اعتبار ارائه دهنده هوش مصنوعی)

روش توصیه شده برای پیکربندی ارائه دهندگان هوش مصنوعی:

```
1. Open Notebook را در مرورگر خود باز کنید
2. به تنظیمات → کلیدهای API بروید
3. روی «افزودن اعتبار» کلیک کنید
4. ارائه دهنده را انتخاب کنید، کلید API را وارد کنید
5. روی Save و سپس Test Connection کلیک کنید
6. روی Discover Models → Register Models کلیک کنید
```

بدون ویرایش فایل، بدون راه اندازی مجدد. اعتبارنامه ها به صورت ایمن (رمزگذاری شده) در پایگاه داده ذخیره می شوند.

→ **[راهنمای کامل: پیکربندی API](../3-USER-GUIDE/api-configuration.md)**

### روش 2: ویرایش فایل «.env» (تنظیمات زیرساخت)

برای تنظیمات پایگاه داده، شبکه و کلید رمزگذاری:

``باش
1. env را در ویرایشگر خود باز کنید
2. OPEN_NOTEBOOK_ENCRYPTION_KEY و متغیرهای پایگاه داده را تنظیم کنید
3. ذخیره کنید
4. راه اندازی مجدد خدمات
```

### روش 3: تنظیم محیط Docker (استقرار)

``باش
# در docker-compose.yml:
خدمات:
  api:
    محیط زیست:
      - OPEN_NOTEBOOK_ENCRYPTION_KEY=کلید مخفی من
      - API_URL=https://your-domain.com
```

## تأیید

پس از پیکربندی، بررسی کنید که کار می کند:

```
1. نوت بوک خود را باز کنید
2. به Settings → Models بروید
3. باید ارائه دهنده پیکربندی شده خود را ببینید
4. یک سوال ساده چت را امتحان کنید
5. اگر پاسخ داد، پیکربندی درست است!
```

## اشتباهات رایج

| اشتباه | مشکل | رفع |
|---------|---------|-----|
| هیچ اعتبار پیکربندی نشده | مدل های موجود نیست | افزودن اعتبار در تنظیمات → کلیدهای API |
| کلید رمزگذاری وجود ندارد | اطلاعات کاربری ذخیره نمی شود | تنظیم OPEN_NOTEBOOK_ENCRYPTION_KEY |
| آدرس پایگاه داده اشتباه است | نمی توان API را راه اندازی کرد | فرمت SURREAL_URL |
| Expose پورت 5055 | "نمی توان به سرور متصل شد" | نمایش 5055 در docker-compose || اشتباه تایپی در env var | تنظیمات نادیده گرفته شد | بررسی املا (حساس به حروف بزرگ و کوچک!) |
| راه اندازی مجدد نشود | پیکربندی قدیمی هنوز استفاده می شود | راه اندازی مجدد خدمات پس از تغییرات env |
