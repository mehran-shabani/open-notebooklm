# یکپارچه سازی پروتکل زمینه مدل (MCP).

Open Notebook را می‌توان با استفاده از **پروتکل زمینه مدل (MCP)** در جریان‌های کاری هوش مصنوعی شما ادغام کرد و امکان دسترسی مستقیم به نوت‌بوک‌ها، منابع و عملکرد چت را از دستیاران هوش مصنوعی مانند Claude Desktop و VS Code فراهم می‌کند.

## MCP چیست؟

[Model Context Protocol] (https://modelcontextprotocol.io) یک استاندارد باز است که به برنامه‌های هوش مصنوعی اجازه می‌دهد به طور ایمن به منابع و ابزارهای داده خارجی متصل شوند. با سرور Open Notebook MCP، می توانید:

- 📚 **دسترسی به نوت بوک های خود** مستقیماً از Claude Desktop یا VS Code
- 🔍 ** بدون ترک دستیار هوش مصنوعی خود، محتوای تحقیقاتی خود را جستجو کنید
- 💬 ** جلسات چت ** با تحقیقات خود به عنوان زمینه ایجاد و مدیریت کنید
- 📝 ** یادداشت ها و بینش ها را در لحظه ایجاد کنید
- 🤖 **گردش های کاری را خودکار** با استفاده از API کامل Open Notebook

## راه اندازی سریع

### برای کلود دسکتاپ

1. ** سرور MCP را نصب کنید ** (به طور خودکار از PyPI):

   ``باش
   # بدون نیاز به نصب دستی! کلود دسکتاپ از uvx برای اجرای خودکار آن استفاده می کند
   ```

2. **پیکربندی Claude Desktop**:

   **macOS/Linux**: ویرایش `~/Library/Application Support/Claude/claude_desktop_config.json`

   ``json
   {
     "mcpServers": {
       "نوت بوک باز": {
         "command": "uvx"،
         "args": ["open-notebook-mcp"]،
         "env": {
           "OPEN_NOTEBOOK_URL": "http://localhost:5055"،
           "OPEN_NOTEBOOK_PASSWORD": "your_password_here"
         }
       }
     }
   }
   ```

   **ویندوز**: ویرایش `%APPDATA%\Claude\claude_desktop_config.json`

   ``json
   {
     "mcpServers": {
       "نوت بوک باز": {
         "command": "uvx"،
         "args": ["open-notebook-mcp"]،
         "env": {
           "OPEN_NOTEBOOK_URL": "http://localhost:5055"،
           "OPEN_NOTEBOOK_PASSWORD": "your_password_here"
         }
       }
     }
   }
   ```

3. **کلود دسکتاپ را راه اندازی مجدد کنید** و شروع به استفاده از نوت بوک خود در مکالمات کنید!

### برای VS Code (Cline و دیگر برنامه‌های افزودنی سازگار با MCP)

به تنظیمات VS Code یا «.vscode/mcp.json» خود اضافه کنید:

``json
{
  "سرورها": {
    "نوت بوک باز": {
      "command": "uvx"،
      "args": ["open-notebook-mcp"]،
      "env": {
        "OPEN_NOTEBOOK_URL": "http://localhost:5055"،
        "OPEN_NOTEBOOK_PASSWORD": "your_password_here"
      }
    }
  }
}
```

## پیکربندی

- **OPEN_NOTEBOOK_URL**: URL به API Notebook Open شما (پیش‌فرض: "http://localhost:5055")
- **OPEN_NOTEBOOK_PASSWORD**: اختیاری - فقط در صورتی لازم است که محافظت از رمز عبور را فعال کرده باشید### برای سرورهای راه دور

اگر نمونه Open Notebook شما روی یک سرور راه دور اجرا می شود، URL را مطابق با آن به روز کنید:

``json
"OPEN_NOTEBOOK_URL": "http://192.168.1.100:5055"
```

یا با دامنه:

``json
"OPEN_NOTEBOOK_URL": "https://notebook.yourdomain.com/api"
```

## آنچه می توانید انجام دهید

پس از اتصال، می توانید از کلود یا دستیار هوش مصنوعی خود بخواهید:

- "برای اطلاعات در مورد [موضوع] در دفترهای تحقیقاتی من جستجو کنید"_
- _"یادداشت جدیدی برای خلاصه کردن نکات کلیدی مکالمه ما ایجاد کنید"_
- _"لیست تمام دفترچه های من"_
- "شروع یک جلسه چت درباره [منبع یا موضوع خاص]"_
- _"چه منابعی در دفترچه یادداشت [نام نوت بوک] خود دارم؟"_
- _"این پی دی اف را به دفترچه تحقیقاتی من اضافه کنید"_
- _"همه یادداشت ها را در [نام نوت بوک] به من نشان بده"_

سرور MCP دسترسی کامل به قابلیت‌های Open Notebook را فراهم می‌کند و به شما امکان می‌دهد تحقیقات خود را به طور یکپارچه از درون دستیار هوش مصنوعی خود مدیریت کنید.

## ابزارهای موجود

سرور MCP Notebook Open این قابلیت ها را نشان می دهد:

### نوت بوک

- فهرست نوت بوک ها
- جزئیات نوت بوک را دریافت کنید
- ایجاد نوت بوک های جدید
- به روز رسانی اطلاعات نوت بوک
- حذف نوت بوک ها

### منابع

- فهرست منابع در یک دفترچه یادداشت
- دریافت جزئیات منبع
- اضافه کردن منابع جدید (لینک ها، فایل ها، متن)
- به روز رسانی ابرداده منبع
- حذف منابع

### یادداشت ها

- یادداشت ها را در یک دفترچه فهرست کنید
- جزئیات یادداشت را دریافت کنید
- یادداشت های جدید ایجاد کنید
- به روز رسانی یادداشت ها
- حذف یادداشت ها

### چت

- ایجاد جلسات چت
- ارسال پیام به جلسات چت
- تاریخچه چت را دریافت کنید
- لیست جلسات چت

### جستجو

- جستجوی برداری در سراسر محتوا
- جستجوی متن در محتوا
- فیلتر توسط نوت بوک

### مدل ها

- مدل های AI پیکربندی شده را فهرست کنید
- جزئیات مدل را دریافت کنید
- ایجاد تنظیمات مدل
- تنظیمات مدل را به روز کنید

### تنظیمات

- تنظیمات برنامه را دریافت کنید
- تنظیمات را به روز کنید

## عیب یابی

### خطاهای اتصال

1. بررسی کنید که "OPEN_NOTEBOOK_URL" صحیح و قابل دسترسی است2. در صورت استفاده از محافظت از رمز عبور، مطمئن شوید که «OPEN_NOTEBOOK_PASSWORD» به درستی تنظیم شده است
3. برای سرورهای راه دور، مطمئن شوید که پورت 5055 از دستگاه شما قابل دسترسی است
4. در صورت اتصال به سرور راه دور، تنظیمات فایروال را بررسی کنید
