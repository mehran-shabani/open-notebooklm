# پیکربندی امنیتی

از استقرار نوت بوک باز خود با احراز هویت رمز عبور و سخت شدن تولید محافظت کنید.

---

## رمزگذاری کلید API

Open Notebook کلیدهای API ذخیره شده در پایگاه داده را با استفاده از رمزگذاری متقارن Fernet (AES-128-CBC با HMAC-SHA256) رمزگذاری می کند.

### روش های پیکربندی

| روش | مستندات |
|--------|---------------|
| **رابط کاربری تنظیمات** | [راهنمای پیکربندی API](../3-USER-GUIDE/api-configuration.md) |
| **متغیرهای محیطی** | این صفحه (زیر) |

### راه اندازی

کلید رمزگذاری را روی هر رشته مخفی تنظیم کنید:

``باش
# .env یا docker.env
OPEN_NOTEBOOK_ENCRYPTION_KEY = عبارت عبور مخفی من
```

هر رشته ای کار می کند - به طور ایمن از طریق SHA-256 داخلی مشتق می شود. از یک کلمه عبور قوی برای استقرار تولید استفاده کنید.

### اعتبار پیش فرض

| تنظیم | پیش فرض | سطح امنیتی |
|---------|---------|----------------|
| رمز عبور | «نوت بوک-تغییر-من» | فقط توسعه |
| کلید رمزگذاری | **هیچ** (باید پیکربندی شود) | مورد نیاز برای ذخیره سازی کلید API |

**کلید رمزگذاری پیش‌فرض ندارد.** قبل از استفاده از ویژگی پیکربندی کلید API، باید «OPEN_NOTEBOOK_ENCRYPTION_KEY» را تنظیم کنید. بدون آن، رمزگذاری/رمزگشایی کلیدهای API با شکست مواجه خواهد شد.

### پشتیبانی از Docker Secrets

هر دو تنظیمات از اسرار Docker از طریق پسوند «_FILE» پشتیبانی می‌کنند:

`` یامل
محیط زیست:
  - OPEN_NOTEBOOK_PASSWORD_FILE=/run/secrets/app_password
  - OPEN_NOTEBOOK_ENCRYPTION_KEY_FILE=/run/secrets/encryption_key
```

### نکات امنیتی

| سناریو | رفتار |
|----------|----------|
| کلید پیکربندی شد | کلیدهای API رمزگذاری شده با کلید شما |
| هیچ کلیدی پیکربندی نشده است | رمزگذاری/رمزگشایی ناموفق خواهد بود (کلید مورد نیاز است) |
| کلید تغییر کرد | کلیدهای رمزگذاری شده قدیمی غیرقابل خواندن می شوند |
| داده های قدیمی | کلیدهای رمزگذاری نشده هنوز کار می کنند (بازگشت دلپذیر) |

### مدیریت کلید

- **Seep Secret**: هرگز کلید رمزگذاری را به کنترل نسخه متعهد نکنید
- **پشتیبان گیری امن**: کلید را جدا از نسخه های پشتیبان پایگاه داده ذخیره کنید
- **هنوز چرخش وجود ندارد**: تغییر کلید نیاز به ذخیره مجدد همه کلیدهای API دارد
- **در هر استقرار**: هر نمونه باید کلید رمزگذاری مخصوص به خود را داشته باشد

## راه اندازی سریع

### استقرار داکر

`` یامل
# به docker-compose.yml خود اضافه کنید (به سرویس سوررئال‌دبی نیاز دارد، راهنمای نصب را ببینید)خدمات:
  open_notebook:
    تصویر: lfnovo/open_notebook:v1-latest
    pull_policy: همیشه
    محیط زیست:
      - OPEN_NOTEBOOK_ENCRYPTION_KEY=کلید رمزگذاری مخفی شما
      - OPEN_NOTEBOOK_PASSWORD=گذرواژه_ایمن_شما
    # ... بقیه تنظیمات
```

یا با استفاده از فایل محیطی:

``باش
# docker.env
OPEN_NOTEBOOK_ENCRYPTION_KEY=کلید رمزگذاری مخفی شما
OPEN_NOTEBOOK_PASSWORD=گذرواژه_ایمن_شما
```

> **مهم**: کلید رمزگذاری برای ذخیره اطلاعات اعتبار **الزامی** است. بدون آن، نمی‌توانید اعتبار ارائه‌دهنده هوش مصنوعی را از طریق تنظیمات UI ذخیره کنید. اگر کلید رمزگذاری را تغییر دهید یا گم کنید، همه اعتبارنامه های ذخیره شده غیرقابل خواندن می شوند.

### راه اندازی توسعه

``باش
# . env
OPEN_NOTEBOOK_PASSWORD=گذرواژه_ایمن_شما
```

## چگونه کار می کند

### محافظت از Frontend

1. فرم ورود در اولین بازدید ظاهر می شود
2. رمز عبور ذخیره شده در جلسه مرورگر
3. Session تا زمان بسته شدن مرورگر ادامه دارد
4. برای خروج از سیستم، داده های مرورگر را پاک کنید

### حفاظت API

تمام نقاط پایانی API نیاز به احراز هویت دارند:

``باش
# درخواست احراز هویت
curl -H "Authorization: Bearer your_password" \
  http://localhost:5055/api/notebooks

# احراز هویت نشده (شکست خواهد شد)
حلقه http://localhost:5055/api/notebooks
# برمی‌گرداند: {"جزئیات": "هدر مجوز وجود ندارد"}
```

### نقاط پایانی محافظت نشده

اینها بدون احراز هویت کار می کنند:

- `/سلامت` - بررسی سلامت سیستم
- `/docs` - اسناد API
- `/openapi.json` - مشخصات OpenAPI

`` یامل

### پیکربندی فایروال

``باش
# UFW (اوبونتو)
sudo ufw اجازه ssh
sudo ufw اجازه می دهد 80/tcp
sudo ufw اجازه 443/tcp
sudo ufw deny 8502/tcp # دسترسی مستقیم را مسدود کنید
sudo ufw deny 5055/tcp # دسترسی مستقیم API را مسدود کنید
sudo ufw را فعال کنید

# iptables
iptables -A INPUT -p tcp --dport 22 -j ACCEPT
iptables -A INPUT -p tcp --dport 80 -j ACCEPT
iptables -A INPUT -p tcp --dport 443 -j ACCEPT
iptables -A INPUT -p tcp --dport 8502 -j DROP
iptables -A INPUT -p tcp --dport 5055 -j DROP
```

### CORS Origins

API به طور پیش‌فرض (`*`) درخواست‌های متقاطع را از هر منبعی می‌پذیرد. این برای توسعه و راه‌اندازی‌های خود میزبانی متنوع مناسب است، اما برای استقرار تولید در اینترنت توصیه نمی‌شود زیرا هر وب‌سایتی که کاربر بازدید می‌کند می‌تواند درخواست‌های اعتبار متقاطع را برای API شما صادر کند.وقتی «CORS_ORIGINS» تنظیم نشده باشد، API یک هشدار راه‌اندازی را ثبت می‌کند که از شما می‌خواهد آن را پیکربندی کنید.

**برای تولید، «CORS_ORIGINS» را روی مبدأ(های) واقعی ظاهر خود تنظیم کنید:**

``باش
# منشاء واحد
CORS_ORIGINS=https://notebook.example.com

# مبدا چندگانه (با کاما از هم جدا شده اند)
CORS_ORIGINS=https://notebook.example.com,https://admin.example.com
```

**راهنما:**

- همیشه از ریشه های HTTPS در تولید استفاده کنید.
- فقط مبداهای دقیقی را که باید مجاز به فراخوانی API باشند فهرست کنید.
- طرح و پورت (در صورت غیر پیش‌فرض) را شامل شود: «https://example.com»، «http://192.168.1.10:3000».
- برای اعمال تغییرات به یک راه اندازی مجدد API نیاز است.

**پاسخ‌های خطا** (401، 404، 500، و غیره) همچنین به مبداهای پیکربندی شده احترام می‌گذارند - آنها فقط «دسترسی-کنترل-اجازه-منبع» را برای مبداهای مجاز شامل می‌شوند، بنابراین وقتی «CORS_ORIGINS» پیکربندی می‌شود، بدنه‌های خطا به‌صورت متقاطع به بیرون درز نمی‌کنند.

## محدودیت های امنیتی

حفاظت از رمز عبور Open Notebook **کنترل دسترسی اولیه** را فراهم می کند، نه امنیت درجه یک سازمانی:

| ویژگی | وضعیت |
|---------|--------|
| انتقال رمز عبور | متن ساده (از HTTPS استفاده کنید!) |
| ذخیره سازی رمز عبور | در حافظه |
| مدیریت کاربر | رمز عبور واحد برای همه |
| پایان جلسه | هیچ (تا زمان بسته شدن مرورگر) |
| محدود کردن نرخ | هیچکدام |
| ثبت حسابرسی | هیچکدام |

### کاهش خطر

1. **همیشه از HTTPS استفاده کنید** - ترافیک را با TLS رمزگذاری کنید
2. **رمزهای عبور قوی** - 20+ کاراکتر، پیچیده
3. **امنیت شبکه** - فایروال، VPN برای استقرارهای حساس
4. **به روز رسانی منظم ** - ظروف و وابستگی ها را به روز نگه دارید
5. ** نظارت ** - سیاهههای مربوط را برای فعالیت مشکوک بررسی کنید
6. ** پشتیبان گیری ** - پشتیبان گیری منظم از داده ها

## عیب یابی

### رمز عبور کار نمی کند

``باش
# بررسی کنید env var تنظیم شده است
docker exec open-notebook env | grep OPEN_NOTEBOOK_PASSWORD

# سیاهههای مربوط را بررسی کنید
Docker logs open-notebook | grep -i auth

# API را مستقیماً آزمایش کنید
curl -H "Authorization: Bearer your_password" \
  http://localhost:5055/health
```

### 401 خطاهای غیرمجاز

``باش
# قالب هدر را بررسی کنید
curl -v -H "Authorization: Bearer your_password" \
  http://localhost:5055/api/notebooks

# مطابقت رمز عبور را تأیید کنیدecho "طول رمز عبور: $(echo -n $OPEN_NOTEBOOK_PASSWORD | wc -c)"
```

### پس از تنظیم رمز عبور امکان دسترسی وجود ندارد

1. کش مرورگر و کوکی ها را پاک کنید
2. حالت ناشناس/خصوصی را امتحان کنید
3. کنسول مرورگر را برای خطا بررسی کنید
4. بررسی صحیح رمز عبور در محیط

### تست امنیتی

``باش
# بدون رمز عبور (باید خراب شود)

  http://localhost:5055/api/notebooks
