# راهنمای راه اندازی اوللاما

Olama مدل های رایگان و محلی هوش مصنوعی را ارائه می دهد که بر روی سخت افزار خود شما اجرا می شوند. این راهنما همه چیزهایی را که باید در مورد راه‌اندازی Ollama با Open Notebook بدانید، شامل سناریوهای مختلف استقرار و پیکربندی‌های شبکه را پوشش می‌دهد.

## چرا اولاما را انتخاب کنید؟

- **🆓 کاملا رایگان **: بدون هزینه API پس از راه اندازی اولیه
- **🔒 حریم خصوصی کامل **: داده های شما هرگز شبکه محلی شما را ترک نمی کند
- **📱 قابلیت آفلاین **: بدون اتصال به اینترنت کار می کند
- **🚀 سریع**: استنتاج محلی بدون تاخیر شبکه
- **🧠 مدل های استدلال **: پشتیبانی از مدل های استدلال پیشرفته مانند DeepSeek-R1
- **💾 تنوع مدل **: دسترسی به صدها مدل منبع باز

## شروع سریع

### 1. اوللاما را نصب کنید

**Linux/macOS:**
``باش
curl -fsSL https://ollama.ai/install.sh | ش
```

**ویندوز:**
دانلود و نصب کنید [ollama.ai](https://ollama.ai/download)

### 2. مدل های مورد نیاز را بکشید

``باش
# مدل زبان (یک یا چند مدل را انتخاب کنید)
olama pull qwen3 # هدف کلی عالی، پارامترهای 7B
olama pull gemma3 # مدل گوگل، عملکرد خوب
olama pull deepseek-r1 # مدل استدلال پیشرفته
olama pull phi4 # مدل کارآمد مایکروسافت

# مدل جاسازی (برای جستجو لازم است)
olama pull mxbai-embed-large # بهترین مدل جاسازی برای اوللاما
```

### 3. Open Notebook را پیکربندی کنید

**از طریق تنظیمات UI (توصیه می شود):**
1. به **تنظیمات** → **کلیدهای API** بروید
2. روی **Add Credential** → Select **Ollama** کلیک کنید
3. URL پایه را وارد کنید (برای URL صحیح به [پیکربندی شبکه] (#network-configuration-guide) در زیر مراجعه کنید)
4. روی **ذخیره** و سپس روی **Test Connection** کلیک کنید.
5. روی **Discover Models** → **Register Models** کلیک کنید

** میراث (منسوخ شده) - متغیرهای محیطی:**
``باش
# برای نصب محلی:
صادرات OLLAMA_API_BASE=http://localhost:11434
# برای نصب داکر:
صادرات OLLAMA_API_BASE=http://host.docker.internal:11434
```

> **نکته**: متغیر محیطی «OLLAMA_API_BASE» منسوخ شده است. Ollama را از طریق تنظیمات → کلیدهای API پیکربندی کنید.

## راهنمای پیکربندی شبکه

هنگام افزودن یک اعتبارنامه Ollama در **تنظیمات → کلیدهای API**، باید URL پایه صحیح را وارد کنید. URL صحیح به سناریوی استقرار شما بستگی دارد:

### سناریوی 1: نصب محلی (همان ماشین)

وقتی هم Open Notebook و هم Ollama مستقیماً روی دستگاه شما اجرا می شوند:

**نشانی وب پایه برای وارد کردن در تنظیمات → کلیدهای API:** `http://localhost:11434`

جایگزین: «http://127.0.0.1:11434» (اگر مشکلات حل DNS با لوکال هاست دارید استفاده کنید)### سناریوی 2: نوت بوک را در Docker، Olama در هاست باز کنید

وقتی Open Notebook در Docker اجرا می‌شود اما Ollama روی دستگاه میزبان شما اجرا می‌شود:

**نشانی وب پایه برای وارد کردن در تنظیمات → کلیدهای API:** `http://host.docker.internal:11434`

**⚠️ مهم: اولاما باید اتصالات خارجی را بپذیرد:**
``باش
# Ollama را با دسترسی خارجی فعال کنید
صادرات OLLAMA_HOST=0.0.0.0:11434
اللاما خدمت کنید
```

**⚠️ کاربران لینوکس: پیکربندی اضافی مورد نیاز است!**

در لینوکس، «host.docker.internal» به‌طور خودکار مانند macOS/Windows حل نمی‌شود. شما باید "extra_hosts" را به docker-compose.yml خود اضافه کنید:

`` یامل
# به docker-compose.yml خود اضافه کنید (به سرویس سوررئال‌دبی نیاز دارد، راهنمای نصب را ببینید)
خدمات:
  open_notebook:
    تصویر: lfnovo/open_notebook:v1-latest
    # ... تنظیمات دیگر ...
    extra_hosts:
      - "host.docker.internal:host-gateway"
```

بدون این، شما خطاهای اتصال مانند:
```
httpcore.ConnectError: [Errno -2] نام یا سرویس مشخص نیست
```

**چرا "host.docker.internal"؟**
- کانتینرهای Docker نمی توانند به "localhost" در میزبان دسترسی پیدا کنند
- «host.docker.internal» نام میزبان ویژه Docker برای دستگاه میزبان است
- موجود در Docker Desktop برای Mac/Windows. **به "هاست_اضافی" در لینوکس نیاز دارد**

**چرا `OLLAMA_HOST=0.0.0.0:11434`؟**
- به طور پیش فرض، Olama فقط به localhost متصل می شود و اتصالات خارجی را رد می کند
- کانتینرهای داکر حتی زمانی که روی یک دستگاه کار می کنند "خارجی" در نظر گرفته می شوند
- تنظیم «OLLAMA_HOST=0.0.0.0:11434» امکان اتصال از کانتینرهای Docker را فراهم می کند

### سناریوی 3: هر دو در Docker (Same Compose)

هنگامی که هر دو Open Notebook و Olama در یک پشته Docker Compose اجرا می شوند:

** URL پایه برای وارد کردن در تنظیمات → کلیدهای API:** `http://ollama:11434`

**Docker Compose مثال:**

`` یامل
# به سرویس surrealdb نیاز دارد - تنظیمات پایه کامل را ببینید:
# https://github.com/lfnovo/open-notebook/blob/main/docker-compose.yml
خدمات:
  دفترچه یادداشت باز:
    تصویر: lfnovo/open_notebook:v1-latest
    pull_policy: همیشه
    پورت ها:
      - "8502:8502"
      - "5055:5055"
    محیط زیست:
      - OPEN_NOTEBOOK_ENCRYPTION_KEY=تغییر-me-to-a-secret-string
    حجم ها:
      - ./notebook_data:/app/data
    بستگی_به:
      - الما

  اولاما:
    تصویر: olama/olama:جدیدترین
    پورت ها:
      - "11434:11434"
    حجم ها:
      - olama_data:/root/.ollama
    # اختیاری: پشتیبانی از GPU
    استقرار:
      منابع:
        رزروها:
          دستگاه ها:
            - درایور: nvidia
              تعداد: 1
              قابلیت ها: [gpu]

حجم ها:
  olama_data:
```### سناریوی 4: سرور Ollama از راه دور

وقتی اولاما روی دستگاه دیگری در شبکه شما اجرا می شود:

** URL پایه برای وارد کردن در تنظیمات → کلیدهای API:** «http://192.168.1.100:11434» (با IP سرور Ollama خود جایگزین کنید)

**نکته امنیتی:** فقط در شبکه های قابل اعتماد از آن استفاده کنید. اولاما احراز هویت داخلی ندارد.

### سناریوی 5: اولاما با پورت سفارشی

اگر Ollama را برای استفاده از پورت دیگری پیکربندی کرده اید:

``باش
# اوللاما را در پورت سفارشی شروع کنید
OLLAMA_HOST=0.0.0.0:8080 سرو اوللاما
```

** URL پایه برای وارد کردن در تنظیمات → کلیدهای API:** `http://localhost:8080`

**⚠️ مهم: نام مدل ها باید دقیقاً با خروجی «فهرست olama» مطابقت داشته باشد**

**مرحله 1: نام دقیق مدل را دریافت کنید**
``باش
لیست اولاما
```

خروجی نمونه:
```
NAME ID SIZE MODIFIED
mxbai-embed-large:last 468836162de7 669 مگابایت 7 دقیقه پیش

**مرحله 2: هنگام اضافه کردن مدل در Open Notebook از نام دقیق استفاده کنید**

|-----------|----------|

3. **نام دقیق** را از 'ollama list' وارد کنید
4. ارائه دهنده را انتخاب کنید: "ollama".
5. نوع را انتخاب کنید: «زبان» (برای چت) یا «جاسازی» (برای جستجو)
6. مدل را ذخیره کنید

### مسائل رایج

**1. "Ollama در دسترس نیست" در Open Notebook**

**بررسی کنید که اولاما در حال اجرا است:**
``باش
حلقه http://localhost:11434/api/tags
```

** تأیید اعتبار پیکربندی شده است:**
**تنظیمات → کلیدهای API** را برای اعتبارنامه اوللاما با URL پایه صحیح بررسی کنید.

**⚠️ مهم: فعال کردن اتصالات خارجی (متداول ترین راه حل):**
``باش
# اگر Open Notebook در Docker یا دستگاه دیگری اجرا شود،
# Ollama باید به همه اینترفیس ها متصل شود، نه فقط localhost
صادرات OLLAMA_HOST=0.0.0.0:11434
اللاما خدمت کنید
```
> **چرا این مورد نیاز است:** به طور پیش فرض، Olama فقط اتصالات "localhost" را می پذیرد (127.0.0.1). وقتی Open Notebook در Docker یا دستگاه دیگری اجرا می‌شود، نمی‌تواند به Ollama دسترسی پیدا کند مگر اینکه «OLLAMA_HOST=0.0.0.0:11434» را برای پذیرش اتصالات خارجی پیکربندی کنید.

**Ollama را دوباره راه اندازی کنید:**
``باش
# Linux/macOS
sudo systemctl راه اندازی مجدد olama
# یا
اللاما خدمت کنید

# ویندوز
# راه اندازی مجدد از سینی سیستم یا خدمات
```

**2. مشکلات شبکه داکر **

**از داخل ظرف نوت بوک باز، اولاما را تست کنید:**

**اگر این مورد در لینوکس** با "نام یا سرویس شناخته نشده" انجام نشد، باید "extra_hosts" را به docker-compose.yml خود اضافه کنید. بخش [عیب‌یابی خاص Docker] (#docker-specific-troubleshooting) را در زیر ببینید.

**6. "پیام ارسال نشد" در چت**

**علامت:** چت اعلان نان تست "ارسال پیام ناموفق" را نشان می دهد. گزارش‌ها ممکن است نشان دهند:
```
خطا در اجرای چت: مدل یک LanguageModel نیست: هیچ
```

**علل (به ترتیب احتمال):**

1. **عدم تطابق نام مدل**: نام مدل در دفترچه یادداشت باز دقیقاً با «فهرست ollama» مطابقت ندارد.
2. **هیچ مدل پیش فرض پیکربندی نشده است**: شما یک مدل چت پیش فرض را در تنظیمات → مدل ها تنظیم نکرده اید
3. **مدل حذف شد**: شما مدل را از Ollama حذف کردید اما پیش فرض های Open Notebook را به روز نکردید
4. ** رکورد مدل حذف شده**: مدل از Open Notebook حذف شد اما همچنان به عنوان پیش فرض تنظیم شده است

**راه حل:**

**بررسی 1: بررسی کنید نام مدل دقیقا مطابقت داشته باشد**
``باش
# نام مدل های دقیق را از اولاما دریافت کنید
لیست اولاما

# با آنچه در Open Notebook پیکربندی شده است مقایسه کنید
# به تنظیمات → مدل‌ها بروید و بررسی کنید که نام‌ها دقیقاً مطابقت دارند
```

**بررسی 2: بررسی مدل های پیش فرض تنظیم شده اند**
1. به **تنظیمات → مدل ها** بروید
2. به بخش **مدل های پیش فرض** بروید
3. مطمئن شوید که **مدل چت پیش فرض** مقداری انتخاب شده است
4. اگر خالی است، یک مدل زبان موجود را انتخاب کنید

**بررسی 3: بازخوانی پس از تغییرات**
اگر مدل‌هایی را در Ollama اضافه یا حذف کرده‌اید:
1. صفحه Open Notebook را بازخوانی کنید
2. به Settings → Models بروید
3. مدل‌های گمشده را با نام‌های دقیق از «فهرست ollama» دوباره اضافه کنید
4. در صورت نیاز دوباره مدل های پیش فرض را انتخاب کنید

**بررسی 4: مدل را مستقیماً آزمایش کنید**
``باش
# تأیید کنید که Olama می تواند از این مدل استفاده کند
olama run gemma3:12b "سلام، دنیا"
```

### عیب یابی خاص Docker**1. لینوکس: «host.docker.internal» حل نمی‌شود (متداول‌ترین)**

اگر خطاهای «نام یا سرویس نامشخص» را در لینوکس می‌بینید، «extra_hosts» را به docker-compose.yml خود اضافه کنید:

`` یامل
# به docker-compose.yml خود اضافه کنید (به سرویس سوررئال‌دبی نیاز دارد، راهنمای نصب را ببینید)
خدمات:
  open_notebook:
    تصویر: lfnovo/open_notebook:v1-latest
    extra_hosts:
      - "host.docker.internal:host-gateway"
    محیط زیست:
    # ... بقیه تنظیمات شما
```

سپس در **تنظیمات → کلیدهای API**، از URL پایه استفاده کنید: «http://host.docker.internal:11434».

این «host.docker.internal» را به IP دستگاه میزبان شما نشان می دهد. macOS/Windows Docker Desktop این کار را به صورت خودکار انجام می دهد، اما لینوکس به پیکربندی صریح نیاز دارد.

**2. شبکه میزبان در لینوکس (جایگزین):**
``باش
# اگر host.docker.internal کار نمی کند از شبکه میزبان استفاده کنید
docker run --network host lfnovo/open_notebook:v1-latest # فقط برای تست سریع
```
سپس در **تنظیمات → کلیدهای API**، از URL پایه استفاده کنید: «http://localhost:11434».

**3. شبکه پل سفارشی:**
`` یامل
نسخه: '3.8'
شبکه ها:
  olama_network:
    راننده: پل

خدمات:
  دفترچه یادداشت باز:
    شبکه ها:
      - olama_network
    محیط زیست:
  اولاما:
    شبکه ها:
      - olama_network
```

سپس در **تنظیمات → کلیدهای API**، از URL پایه استفاده کنید: «http://ollama:11434».

**4. مشکلات فایروال:**
``باش
# اجازه پورت Ollama از طریق فایروال
sudo ufw allow 11434
# یا
sudo firewall-cmd --add-port=11434/tcp --permanent
```

## پیکربندی پیشرفته

### متغیرهای محیطی

``باش
# پیکربندی سرور Ollama
صادرات OLLAMA_HOST=0.0.0.0:11434 # اتصال به همه اینترفیس هاصادرات OLLAMA_KEEP_ALIVE=5m # مدل ها را در حافظه نگه دارید
صادرات OLLAMA_MAX_LOADED_MODELS=3 # حداکثر مدل همزمان

صادرات OLLAMA_NUM_PARALLEL=4 # رسیدگی به درخواست موازی
صادرات OLLAMA_FLASH_ATTENTION=1 # فعال کردن توجه فلاش (در صورت پشتیبانی)

# پیکربندی نوت بوک را باز کنید (به جای آن از طریق تنظیمات → کلیدهای API پیکربندی کنید)
# OLLAMA_API_BASE=http://localhost:11434 # منسوخ شده — از تنظیمات رابط کاربری استفاده کنید
```

### پیکربندی SSL (گواهینامه های خودامضا)

اگر Ollama را پشت یک پروکسی معکوس با گواهینامه های SSL خود امضا شده اجرا می کنید (به عنوان مثال، Caddy، nginx با گواهی های سفارشی)، ممکن است با خطاهای تأیید SSL روبرو شوید:

```
[SSL: CERTIFICATE_VERIFY_FAILED] تأیید گواهی ناموفق بود: دریافت گواهی صادرکننده محلی ممکن نیست
```

**راه حل:**

**گزینه 1: استفاده از بسته CA سفارشی (توصیه می شود)**
``باش
# به فایل گواهی CA خود اشاره کنید
صادرات ESPERANTO_SSL_CA_BUNDLE=/path/to/your/ca-bundle.pem
```

**گزینه 2: تأیید SSL را غیرفعال کنید (فقط برای توسعه)**
``باش
# هشدار: فقط در محیط های توسعه قابل اعتماد استفاده کنید
صادرات ESPERANTO_SSL_VERIFY=نادرست
```

**نمونه Docker Compose با پیکربندی SSL:**
`` یامل
# به docker-compose.yml خود اضافه کنید (به سرویس سوررئال‌دبی نیاز دارد، راهنمای نصب را ببینید)
خدمات:
  دفترچه یادداشت باز:
    تصویر: lfnovo/open_notebook:v1-latest
    pull_policy: همیشه
    محیط زیست:
      - OPEN_NOTEBOOK_ENCRYPTION_KEY=تغییر-me-to-a-secret-string
      # گزینه 1: بسته CA سفارشی (اگر اولاما از SSL خودامضا شده استفاده کند)
      - ESPERANTO_SSL_CA_BUNDLE=/certs/ca-bundle.pem
      # گزینه 2: تأیید صحت را غیرفعال کنید (فقط توسعه دهنده)
      # - ESPERANTO_SSL_VERIFY=نادرست
    حجم ها:
      - /path/to/your/ca-bundle.pem:/certs/ca-bundle.pem:ro
```

> **نکته امنیتی:** غیرفعال کردن تأیید SSL شما را در معرض حملات Man-in-the-Middle قرار می دهد. همیشه استفاده از بسته CA سفارشی را در محیط های تولید ترجیح دهید.

## بهترین شیوه ها

### امنیت

1. **امنیت شبکه:**
   - Ollama را فقط در شبکه های قابل اعتماد اجرا کنید
   - از قوانین فایروال برای محدود کردن دسترسی استفاده کنید
   - VPN را برای دسترسی از راه دور در نظر بگیرید

2. **تأیید مدل:**
   - مدل ها را فقط از منابع قابل اعتماد بکشید
   - در صورت امکان بررسی مدل ها را بررسی کنید

   - محدودیت های حافظه و CPU را در تولید تنظیم کنید
   - استفاده از منابع را به طور منظم نظارت کنید

1. **انتخاب مدل:**
   - از اندازه مدل مناسب برای سخت افزار خود استفاده کنید

   - مدل های استفاده نشده را به طور مرتب حذف کنید
   - نظارت بر منابع سیستم

3. **بهینه سازی شبکه:**
   - از شبکه های محلی برای تأخیر بهتر استفاده کنید
