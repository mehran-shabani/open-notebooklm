# بررسی امنیتی - رابط کاربری پیکربندی API

## تاریخ: 2026-01-27 (به روز رسانی: 2026-01-28)
## داور: حسابرسی امنیتی

---

## خلاصه

بررسی امنیتی اجرای مدیریت کلید API برای Open Notebook. پیاده سازی از رویکرد اول پایگاه داده با متغیرهای محیطی استفاده می کند.

---

## رمزگذاری

| مورد | وضعیت | یادداشت ها |
|------|--------|-------|
| رمزگذاری Fernet پیاده سازی شد | گذر | «open_notebook/utils/encryption.py» از AES-128-CBC + HMAC-SHA256 استفاده می کند |
| کلیدهای رمزگذاری شده قبل از ذخیره سازی DB | گذر | «encrypt_value()» در ذخیره | اعمال شد
| کلیدها فقط در صورت نیاز رمزگشایی می شوند | گذر | «decrypt_value()» هنگام خواندن | فراخوانی شد
| کلید رمزگذاری مورد نیاز | گذر | بدون کلید پیش فرض؛ ValueError در صورت عدم پیکربندی |
| پشتیبانی از اسرار داکر | گذر | الگوی پسوند «_FILE» پشتیبانی می شود |
| مستند شده در .env.example | گذر | کلید رمزگذاری مستند شده |

## توصیه های باقی مانده

### بهبودهای آینده

1. **محدودیت نرخ** - افزودن محدودیت نرخ در نقاط پایانی `/credentials/*`
2. **ویژگی های تکمیل خودکار** - "autocomplete="new-password"" را به همه ورودی های رمز عبور اضافه کنید
3. **نمایش 4 کاراکتر آخر ** - نمایش فرمت `********xxxx` برای شناسایی کلید
4. ** ورود به سیستم حسابرسی ** - ورود کلید API تغییرات با مهر زمان

## نتیجه گیری

اجرای رابط کاربری تنظیمات API الزامات امنیتی را برآورده می کند:

- کلیدهای API در حالت استراحت با استفاده از Fernet رمزگذاری شده اند (کلید باید به صراحت پیکربندی شود)
- کلیدها هرگز به قسمت جلو بازنگشتند
- اعتبار سنجی URL از حملات SSRF جلوگیری می کند
- اسرار داکر برای استقرار تولید پشتیبانی می شود

**وضعیت بررسی: PASS**
