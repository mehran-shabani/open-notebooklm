# سوالات متداول

سوالات متداول در مورد استفاده، پیکربندی و بهترین روش‌ها از Notebook Open.

---

## استفاده عمومی

### Open Notebook چیست؟

Open Notebook یک جایگزین منبع باز و متمرکز بر حریم خصوصی برای Notebook LM گوگل است. به شما اجازه می دهد:
- ایجاد و مدیریت دفترهای پژوهشی
- با استفاده از هوش مصنوعی با اسناد خود چت کنید
- از محتوای خود پادکست تولید کنید
- در تمام منابع خود با جستجوی معنایی جستجو کنید
- محتوای خود را متحول و تجزیه و تحلیل کنید

### چه تفاوتی با Google Notebook LM دارد؟

**حریم خصوصی**: داده های شما به طور پیش فرض محلی می ماند. فقط ارائه دهندگان هوش مصنوعی منتخب شما سؤالات را دریافت می کنند.
**انعطاف پذیری**: پشتیبانی از 17+ ارائه دهنده هوش مصنوعی (OpenAI، Anthropic، Google، مدل های محلی، و غیره)
**سفارشی سازی**: منبع باز، بنابراین می توانید عملکرد را تغییر داده و گسترش دهید
**کنترل**: داده ها، مدل ها و پردازش های خود را کنترل می کنید

### آیا می توانم از Open Notebook به صورت آفلاین استفاده کنم؟

**جزئی**: برنامه به صورت محلی اجرا می شود، اما برای موارد زیر به اینترنت نیاز دارد:
- تماس‌های API مدل AI (مگر اینکه از مدل‌های محلی مانند اوللاما استفاده کنید)
- خراش دادن محتوای وب

**کاملاً آفلاین**: با مدل های محلی (Ollama) برای عملکردهای اولیه امکان پذیر است.

### چه نوع فایلی پشتیبانی می شود؟

** اسناد **: PDF، DOCX، TXT، Markdown
**محتوای وب**: URL ها، ویدئوهای یوتیوب
**رسانه**: MP3، WAV، M4A (صوتی)، MP4، AVI، MOV (ویدئو)
**سایر **: ورودی متن مستقیم، CSV، فایل های کد

### هزینه آن چقدر است؟

**نرم افزار**: رایگان (متن باز)
**هزینه های AI API**: پرداخت به ازای استفاده به ارائه دهندگان:
- OpenAI: ~ 0.50-5 دلار در هر 1 میلیون توکن
- Anthropic: ~ 3-75 دلار در هر 1 میلیون توکن
- Google: اغلب سطح رایگان در دسترس است
- مدل های محلی: پس از راه اندازی اولیه رایگان است

**هزینه های معمولی ماهانه**: 5-50 دلار برای استفاده متوسط.

## مدل ها و ارائه دهندگان هوش مصنوعی

### کدام ارائه دهنده هوش مصنوعی را انتخاب کنم؟

**برای مبتدیان**: OpenAI (قابل اعتماد، مستند)
**برای حفظ حریم خصوصی**: مدل های محلی (Ollama) یا ارائه دهندگان اروپایی (Mistral)
**برای بهینه سازی هزینه**: Groq، Google (سطح رایگان)، یا OpenRouter
**برای زمینه طولانی**: Anthropic (200 هزار توکن) یا Google Gemini (1 میلیون توکن)

### آیا می توانم از چندین ارائه دهنده استفاده کنم؟

**بله**: ارائه دهندگان مختلف را برای کارهای مختلف پیکربندی کنید:
- OpenAI برای چت
- گوگل برای جاسازی
- ElevenLabs برای تبدیل متن به گفتار
- آنتروپیک برای استدلال پیچیده

### بهترین ترکیب مدل ها کدامند؟

**بودجه پسند**:
- زبان: "gpt-4o-mini" (OpenAI) یا "deepseek-chat".
- جاسازی: `text-embedding-3-small` (OpenAI)

**کیفیت بالا**:
- زبان: "claude-3-5-sonnet" (Anthropic) یا "gpt-4o" (OpenAI)
- جاسازی: `text-embedding-3-large` (OpenAI)

**مربوط به حریم خصوصی**:
- زبان: مدل های محلی اولاما (mistral، llama3)
- Embedding: مدل های تعبیه محلی

### چگونه هزینه های هوش مصنوعی را بهینه کنم؟

**انتخاب مدل**:
- از مدل های کوچکتر برای کارهای ساده استفاده کنید (gpt-4o-mini، claude-3-5-haiku)
- از مدل های بزرگتر فقط برای استدلال های پیچیده استفاده کنید
- در صورت موجود بودن از سطوح رایگان استفاده کنید

**بهینه سازی استفاده**:
- از زمینه "فقط خلاصه" برای منابع پس زمینه استفاده کنید
- سوالات دقیق تری بپرسید
- از مدل های محلی (Ollama) برای کارهای مکرر استفاده کنید

## مدیریت داده

### داده های من کجا ذخیره می شوند؟

**ذخیره سازی محلی**: به طور پیش فرض، تمام داده ها به صورت محلی ذخیره می شوند:
- پایگاه داده: فایل‌های SurrealDB در «داده_سوررئال/».
- آپلود: فایل‌های موجود در «داده/آپلود/».
- پادکست: صدا در «داده/پادکست/» تولید می شود
- بدون انتقال داده خارجی (به جز ارائه دهندگان AI منتخب)

### چگونه از اطلاعات خود نسخه پشتیبان تهیه کنم؟

``باش
# پشتیبان ایجاد کنید
tar -czf backup-$(date +%Y%m%d).tar.gz data/ surreal_data/

# بازیابی نسخه پشتیبان
tar -xzf backup-20240101.tar.gz
```

### آیا می توانم داده ها را بین دستگاه ها همگام کنم؟

**در حال حاضر**: قابلیت همگام سازی داخلی وجود ندارد.
**راهکار**:
- از ذخیره سازی شبکه مشترک برای دایرکتوری های داده استفاده کنید
- پشتیبان گیری/بازیابی دستی بین دستگاه ها

### اگر یک نوت بوک را حذف کنم چه اتفاقی می افتد؟

**حذف نرم**: نوت بوک ها به عنوان بایگانی شده علامت گذاری می شوند، نه برای همیشه حذف شده اند.
**بازیابی**: نوت بوک های آرشیو شده را می توان از پایگاه داده بازیابی کرد.

## بهترین شیوه ها

### چگونه باید نوت بوک هایم را مرتب کنم؟

- **بر اساس موضوع**: دفترچه های جداگانه برای حوزه های مختلف تحقیقاتی
- **براساس پروژه**: یک دفترچه در هر پروژه یا دوره
- **براساس دوره زمانی**: نوت بوک ماهانه یا فصلی

**اندازه توصیه شده**: 20-100 منبع در هر نوت بوک برای بهترین عملکرد.

### چگونه می توانم بهترین نتایج جستجو را دریافت کنم؟

- از پرس و جوهای توصیفی استفاده کنید ("روش های تجزیه و تحلیل داده ها" نه فقط "داده ها")
- چند اصطلاح مرتبط را با هم ترکیب کنید
- از زبان طبیعی استفاده کنید (همانطور که از یک انسان سؤال می کنید)
- جستجوی متنی (کلمات کلیدی) و جستجوی برداری (مفاهیم) را امتحان کنید

### چگونه می توانم پاسخ های چت را بهبود بخشم؟

- ارائه زمینه: به منابع یا موضوعات خاص ارجاع دهید
- مشخص باشید: به جای سؤالات کلی، سؤالات مفصل بپرسید
- درخواست نقل قول: "پاسخ با نقل قول های صفحه"
- از سؤالات بعدی استفاده کنید: بر اساس پاسخ های قبلی بسازید

### بهترین شیوه های امنیتی چیست؟

- هرگز کلیدهای API را به صورت عمومی به اشتراک نگذارید
- از «OPEN_NOTEBOOK_PASSWORD» برای استقرار عمومی استفاده کنید
- استفاده از HTTPS برای تولید (از طریق پراکسی معکوس)
- تصاویر داکر را به روز نگه دارید
- اگر پشتیبان‌ها حاوی داده‌های حساس هستند، آن‌ها را رمزگذاری کنید

## سوالات فنی

### آیا می توانم از Open Notebook به صورت برنامه ای استفاده کنم؟

**بله**: Open Notebook یک REST API ارائه می دهد:
- اسناد کامل API در "http://localhost:5055/docs".
- پشتیبانی از تمامی قابلیت های رابط کاربری
- احراز هویت از طریق هدر رمز عبور

### آیا می توانم Open Notebook را در مرحله تولید اجرا کنم؟

**بله**: طراحی شده برای مصارف تولیدی با:
- استقرار داکر
- ویژگی های امنیتی (محافظت با رمز عبور)
- نظارت و ثبت گزارش
- پشتیبانی از پروکسی معکوس (nginx، Caddy، Traefik)

### سیستم مورد نیاز چیست؟

**حداقل**:
- 4 گیگابایت رم
- 2 هسته CPU
- 10 گیگابایت فضای دیسک

**توصیه می شود**:
- 8 گیگابایت + رم
- بیش از 4 هسته CPU
- ذخیره سازی SSD
- برای مدل های محلی: 16 گیگابایت رم +، GPU توصیه می شود

## تایم اوت و عملکرد

### چرا خطاهای تایم اوت دریافت می کنم؟

**علل شایع**:
- زمینه بزرگ (منابع بسیار زیاد)
- ارائه دهنده کند هوش مصنوعی
- مدل های محلی روی CPU (آهسته)
- اولین درخواست (بارگیری مدل)

**راه حل**:
``باش
# در env:
API_CLIENT_TIMEOUT=600 # 10 دقیقه برای تنظیمات آهسته
ESPERANTO_LLM_TIMEOUT=180 # 3 دقیقه برای استنتاج مدل
```

### وقفه های زمانی توصیه شده بر اساس راه اندازی:

| راه اندازی | API_CLIENT_TIMEOUT |
|-------|-------------------|
| Cloud API (OpenAI، Anthropic) | 300 (پیش فرض) |
| Ollama محلی با GPU | 600 |
| Ollama محلی با CPU | 1200 |
| ریموت LM Studio | 900 |
