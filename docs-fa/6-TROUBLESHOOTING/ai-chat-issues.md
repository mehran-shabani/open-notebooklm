# مشکلات هوش مصنوعی و چت - پیکربندی و کیفیت مدل

مشکلات مدل های هوش مصنوعی، چت و کیفیت پاسخ.

> **توجه:** Open Notebook اکنون پیام های خطای توصیفی را برای خرابی های ارائه دهنده هوش مصنوعی نشان می دهد. به‌جای «خطای غیرمنتظره‌ای روی داد»، پیام‌های خاصی مانند «احراز هویت ناموفق بود. لطفاً کلید API خود را بررسی کنید» یا «میزان امتیاز فراتر رفته است. لطفاً کمی صبر کنید و دوباره امتحان کنید» را خواهید دید. این پیام‌ها به شما کمک می‌کنند تا مشکلات را سریع‌تر تشخیص داده و برطرف کنید.

---

## خطای "ارسال پیام ناموفق".

**علامت:** چت نان تست "ارسال پیام ناموفق" را نشان می دهد. گزارش ها نشان می دهد:
```
خطا در اجرای چت: مدل یک LanguageModel نیست: هیچ
```

**علت:** هیچ مدل زبان معتبری برای چت پیکربندی نشده است

**راه حل:**

### راه حل 1: تنظیمات مدل پیش فرض را بررسی کنید
```
1. به Settings → Models بروید
2. به بخش "مدل های پیش فرض" بروید
3. بررسی کنید که «مدل چت پیش‌فرض» مدلی انتخاب شده باشد
4. اگر خالی است، یک مدل زبان موجود را انتخاب کنید
5. روی ذخیره کلیک کنید
```

### راه حل 2: تأیید نام مدل (کاربران Ollama)
``باش
# نام مدل های دقیق را دریافت کنید
لیست اولاما

# خروجی نمونه:
# اندازه نام تغییر کرد
# gemma3:12b 8.1 گیگابایت 2 ماه پیش

# نام مدل در Open Notebook باید دقیقا "gemma3:12b" باشد.
# "gemma3" یا "gemma3-12b" نیست
```

### راه حل 3: مدل های گمشده را دوباره اضافه کنید
```
1. به نام مدل های دقیق ارائه دهنده خود توجه کنید
2. به Settings → Models بروید
3. تمام مدل های پیکربندی نادرست را حذف کنید
4. مدل هایی را با نام های دقیق اضافه کنید
5. پیش فرض های جدید را تنظیم کنید
```

### راه حل 4: بررسی کنید که مدل هنوز وجود دارد
``باش
# برای اولاما: بررسی مدل نصب شده است
لیست اولاما

# برای ارائه دهندگان ابر: تأیید کنید کلید API معتبر است
# و شما به مدل دسترسی دارید
```

> **نکته:** این خطا اغلب زمانی رخ می دهد که یک مدل را از Olama حذف می کنید اما فراموش می کنید مدل های پیش فرض را در Open Notebook به روز کنید. همیشه پس از حذف مدل ها، تنظیمات پیش فرض را دوباره پیکربندی کنید.

## "مدل ها در دسترس نیستند" یا "مدل ها نشان داده نمی شوند"

**علامت:** تنظیمات → مدل‌ها خالی نشان داده می‌شوند یا "هیچ مدلی پیکربندی نشده است"

**علت:** هیچ اعتباری پیکربندی نشده است، یا اعتبارنامه دارای کلید API نامعتبر است

**راه حل:**

### راه حل 1: اضافه کردن اعتبار از طریق تنظیمات UI
```
1. به تنظیمات → کلیدهای API بروید
2. روی «افزودن اعتبار» کلیک کنید
3. ارائه دهنده خود را انتخاب کنید (به عنوان مثال OpenAI، Anthropic، Google)
4. کلید API خود را وارد کنید
5. روی Save و سپس Test Connection کلیک کنید
6. روی Discover Models → Register Models کلیک کنید
7. برای تأیید به تنظیمات → Models بروید
```

### راه حل 2: بررسی کلید معتبر است
```
1. به تنظیمات → کلیدهای API بروید
2. روی "Test Connection" در اعتبار خود کلیک کنید
3. اگر "کلید API نامعتبر" را نشان دهد:
   - یک کلید تازه از وب سایت ارائه دهنده دریافت کنید
   - اعتبارنامه را حذف کنید و یک اعتبار جدید ایجاد کنید
```

### راه حل 3: سوئیچ ارائه دهنده
```
1. به تنظیمات → کلیدهای API بروید
2. یک اعتبارنامه برای ارائه دهنده دیگری اضافه کنید
3. تست اتصال → کشف مدل ها → ثبت مدل ها
4. برای انتخاب مدل های ارائه دهنده جدید به Settings → Models بروید
```

## "کلید API نامعتبر" یا "غیر مجاز"

**علامت:** خطا هنگام تلاش برای چت: "کلید API نامعتبر"

**علت:** اعتبارنامه دارای کلید API اشتباه، منقضی شده یا باطل شده است

**راه حل:**

### مرحله 1: اعتبار خود را آزمایش کنید
```
1. به تنظیمات → کلیدهای API بروید
2. روی "Test Connection" در اعتبار خود کلیک کنید
3. اگر شکست خورد، به مرحله 2 بروید
```

### مرحله 2: کلید تازه را دریافت کنید
```
به داشبورد ارائه دهنده بروید:
- OpenAI: https://platform.openai.com/api-keys (با sk-proj- شروع می شود)
- Anthropic: https://console.anthropic.com/ (با sk-ant- شروع می شود)
- گوگل: https://aistudio.google.com/app/apikey (با AIzaSy شروع می شود)

کلید جدید ایجاد کنید و دقیقاً کپی کنید (بدون فاصله اضافی)
```

### مرحله 3: اعتبارنامه را به روز کنید
```
1. به تنظیمات → کلیدهای API بروید
2. اعتبار قدیمی را حذف کنید
3. روی "افزودن اعتبار" → ارائه دهنده را انتخاب کنید
4. کلید جدید را جایگذاری کنید
5. روی Save و سپس Test Connection کلیک کنید
6. در صورت نیاز مدل ها را دوباره کشف و ثبت کنید
```

### مرحله 4: در UI تأیید کنید
```
1. به Settings → Models بروید
2. بررسی مدل های موجود است
3. یک چت آزمایشی را امتحان کنید
```

## چت پاسخ‌های عمومی/بد را نشان می‌دهد

**علامت:** پاسخ های هوش مصنوعی کم عمق، عمومی یا اشتباه هستند

**علت:** زمینه بد، سوال مبهم یا مدل اشتباه

**راه حل:**

### راه حل 1: Context را بررسی کنید
```
1. در چت، روی «انتخاب منابع» کلیک کنید
2. بررسی کنید که منابع مورد نظرتان بررسی شده باشند
3. آنها را روی «محتوای کامل» (نه «فقط خلاصه») تنظیم کنید.
4. روی «ذخیره» کلیک کنید
5. دوباره چت را امتحان کنید
```

### راه حل 2: سوال بهتری بپرسید
```
بد: "نظرت چیه؟"
خوب: "بر اساس روش مقاله، 3 محدودیت چیست؟"

بد: "درباره X به من بگو"
خوب: "X را در 3 نقطه با نقل قول های صفحه خلاصه کنید"
```

### راه حل 3: از مدل قوی تر استفاده کنید
```
OpenAI:
  جریان: gpt-4o-mini → تغییر به: gpt-4o

آنتروپیک:
  فعلی: claude-3-5-haiku → تغییر به: claude-3-5-sonnet

برای تغییر:
1. تنظیمات → مدل ها
2. مدل را انتخاب کنید
3. دوباره چت را امتحان کنید
```

### راه حل 4: اضافه کردن منابع بیشتر
```
اگر: "پاسخ ناقص به نظر می رسد"
سعی کنید: منابع مرتبط بیشتری را برای ارائه زمینه اضافه کنید
```

## چت بسیار کند است

** علامت: ** پاسخ های چت چند دقیقه طول می کشد

**علت:** زمینه بزرگ، مدل کند یا API بیش از حد بارگذاری شده

**راه حل:**

### راه حل 1: از مدل سریعتر استفاده کنید
``باش
سریعترین: Groq (هر مدلی)
سریع: OpenAI gpt-4o-mini
متوسط: Anthropic claude-3-5-haiku
آهسته: Anthropic claude-3-5-sonnet

جابه‌جایی: تنظیمات → مدل‌ها
```

### راه حل 2: کاهش زمینه
```
1. چت → منابع را انتخاب کنید
2. تیک منابعی را که نیاز ندارید بردارید
3. یا برای منابع پس‌زمینه به «فقط خلاصه» بروید
4. ذخیره کنید و دوباره امتحان کنید
```

### راه حل 3: افزایش Timeout
``باش
# در env:
API_CLIENT_TIMEOUT=600 # 10 دقیقه

# راه اندازی مجدد:
docker compose راه اندازی مجدد
```

### راه حل 4: بار سیستم را بررسی کنید
``باش
# ببینید آیا API بیش از حد بارگذاری شده است:
آمار داکر

# اگر CPU > 80٪ یا حافظه > 90٪:
# کاهش: SURREAL_COMMANDS_MAX_TASKS=2
# راه اندازی مجدد: داکر نوشتن مجدد راه اندازی مجدد
```

## چت تاریخ را به یاد نمی آورد

** علامت: ** هر پیام به عنوان جداگانه در نظر گرفته می شود، بدون زمینه بین سوالات

**علت:** سابقه چت ذخیره نشد یا چت جدید شروع شد

**راه حل:**

```
1. مطمئن شوید که در همان چت هستید (نه چت جدید)
2. عنوان چت را در بالا بررسی کنید
3. اگر خالی است، چت جدید را با عنوان شروع کنید
4. هر چت با نام تاریخچه خود را حفظ می کند
5. اگر چت جدید را شروع کنید، سابقه جدا است
```

## "از حد مجاز بیشتر شد"

**علامت:** خطا: "از حد مجاز بیشتر شد" یا "تعداد زیادی درخواست"

** علت: ** به محدودیت نرخ API ارائه دهنده ضربه بزنید

**راه حل:**

### برای ارائه دهندگان ابر (OpenAI، Anthropic، و غیره)

**فوری:**
- 1-2 دقیقه صبر کنید
- دوباره امتحان کن

**کوتاه مدت:**
- از مدل ارزانتر/کوچکتر استفاده کنید
- کاهش عملیات همزمان
- درخواست ها را خالی کنید

**بلند مدت:**
- اکانت خود را ارتقا دهید
- به ارائه دهنده مختلف تغییر دهید
- از Ollama (محلی، بدون محدودیت) استفاده کنید

### وضعیت حساب را بررسی کنید
```
OpenAI: https://platform.openai.com/account/usage/overview
Anthropic: https://console.anthropic.com/account/billing/overview
Google: Google Cloud Console
```

### برای اولاما (محلی)
- بدون محدودیت نرخ
- برای بهترین مدل از «olama pull mistral» استفاده کنید
- در صورت رسیدن به محدودیت های منابع، راه اندازی مجدد

## "طول متن بیشتر شد" یا "محدودیت رمز"

** علامت: ** خطا در مورد تعداد بیش از حد توکن

**علت:** منابع برای مدل خیلی بزرگ هستند

**راه حل:**

### راه حل 1: از مدل با زمینه طولانی تر استفاده کنید
```
فعلی: GPT-4o (128 هزار توکن) → تغییر به: کلود (200 هزار توکن)
فعلی: کلود هایکو (200 هزار نفر) → تغییر به: جمینی (1 میلیون توکن)

برای تغییر: تنظیمات → مدل ها
```

### راه حل 2: کاهش زمینه
```
1. منابع کمتری را انتخاب کنید
2. یا از "Summary Only" به جای "Full Content" استفاده کنید
3. یا اسناد بزرگ را به قطعات کوچکتر تقسیم کنید
```

### راه حل 3: برای اولاما (محلی)
``باش
# از مدل کوچکتر استفاده کنید:
olama pull phi # خیلی کوچک
# به جای: olama pull neuro-chat # Large
```

## "تماس API ناموفق" یا مهلت زمانی

** علامت: ** خطای عمومی API، زمان پاسخ به پایان می رسد

**علت:** API ارائه دهنده از کار افتاده، مشکل شبکه یا کندی سرویس

**راه حل:**

### وضعیت ارائه دهنده را بررسی کنید
```
OpenAI: https://status.openai.com/
Anthropic: وب سایت را بررسی کنید
Google: وضعیت Google Cloud
Groq: وب سایت را بررسی کنید
```

### عملیات را دوباره امتحان کنید
```
1. 30 ثانیه صبر کنید
2. دوباره امتحان کنید
```

### از مدل/ارائه دهنده متفاوت استفاده کنید
```
1. تنظیمات → مدل ها
2. ارائه دهنده های مختلف را امتحان کنید
3. اگر OpenAI پایین است، از Anthropic استفاده کنید
```

### شبکه را بررسی کنید
``باش
# بررسی عملکرد اینترنت:
پینگ google.com

# نقطه پایانی API را مستقیماً آزمایش کنید:
حلقه https://api.openai.com/v1/models \
  -H "مجوز: حامل YOUR_KEY"
```

## پاسخ ها شامل توهم هستند

**علامت:** هوش مصنوعی حقایقی را می سازد که در منابع موجود نیستند

**علت:** منابع در زمینه، یا حدس زدن مدل نیستند

**راه حل:**

### راه حل 1: بررسی زمینه
```
1. در پاسخ روی نقل قول کلیک کنید
2. بررسی منبع در واقع می گوید که
3. اگر نه، منابع در متن نبودند
4. منبع را به متن اضافه کنید و دوباره امتحان کنید
```

### راه حل 2: درخواست نقل قول
```
بپرسید: "این را با نقل قول به صفحات خاص پاسخ دهید"

در صورت درخواست استناد، هوش مصنوعی بیشتر مراقب خواهد بود
```

### راه حل 3: از مدل قوی تر استفاده کنید
```
مدل های ضعیف بیشتر توهم می زنند
تغییر به: GPT-4o یا کلود سونت
```

## هزینه API بالا

**علامت:** صورتحساب های API بالاتر از حد انتظار است

**علت:** استفاده از مدل گران قیمت، زمینه بزرگ، درخواست های زیاد

**راه حل:**

### از مدل ارزانتر استفاده کنید
```
گران قیمت: gpt-4o
ارزان تر: gpt-4o-mini (10 برابر ارزان تر)

گران قیمت: کلود سونت
ارزان تر: کلود هایکو (5 برابر ارزان تر)

Groq: بسیار ارزان اما مدل های کمتر
```

### زمینه را کاهش دهید
```
در چت:
1. منابع کمتری را انتخاب کنید
2. از "Summary Only" برای پس زمینه استفاده کنید
3. سؤالات خاص تری بپرسید
```

### تغییر به اوللاما (رایگان)
``باش
# Olama را نصب کنید
# اجرا: اولاما سرو کنید
# دانلود: olama pull mistral
# مجموعه: OLLAMA_API_BASE=http://localhost:11434
# هزینه: رایگان!
```
