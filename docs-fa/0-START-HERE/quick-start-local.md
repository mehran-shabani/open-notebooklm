# شروع سریع - محلی و خصوصی (5 دقیقه)

با استفاده از Olama با **100% هوش مصنوعی محلی** Open Notebook را اجرا کنید. بدون نیاز به کلیدهای API ابری، کاملا خصوصی.

## پیش نیاز

1. **Docker Desktop** نصب شده است
   - [از اینجا دانلود کنید](https://www.docker.com/products/docker-desktop/)
   - از قبل دارید؟ به مرحله 2 بروید

2. ** LLM محلی ** - یکی را انتخاب کنید:
   - **Ollama** (توصیه می شود): [از اینجا دانلود کنید](https://ollama.ai/)
   - **LM Studio** (جایگزین رابط کاربری گرافیکی): [از اینجا دانلود کنید] (https://lmstudio.ai)

## مرحله 1: تنظیمات خود را انتخاب کنید (1 دقیقه)

### ماشین محلی (همان کامپیوتر)
همه چیز روی دستگاه شما اجرا می شود. برای تست / یادگیری توصیه می شود.

### سرور از راه دور (Raspberry Pi، NAS، Cloud VM)
روی رایانه دیگری اجرا کنید، از رایانه دیگری دسترسی داشته باشید. به پیکربندی شبکه نیاز دارد.

## مرحله 2: ایجاد پیکربندی (1 دقیقه)

یک پوشه جدید «open-notebook-local» ایجاد کنید و این فایل را اضافه کنید:

**docker-compose.yml**:
`` یامل
خدمات:
  surrealdb:
    تصویر: surrealdb/surrealdb:v2
    دستور: start --user root --pass password --bind 0.0.0.0:8000 rocksdb:/mydata/mydatabase.db
    کاربر: ریشه
    پورت ها:
      - "8000:8000"
    حجم ها:
      - ./سوررئال_داده:/mydata

  open_notebook:
    تصویر: lfnovo/open_notebook:v1-latest
    pull_policy: همیشه
    پورت ها:
      - "8502:8502" # رابط کاربری وب (React frontend)
      - "5055:5055" # API (الزامی!)
    محیط زیست:
      # کلید رمزگذاری برای ذخیره اعتبار (الزامی)
      - OPEN_NOTEBOOK_ENCRYPTION_KEY=تغییر-me-to-a-secret-string

      # پایگاه داده (الزامی)
      - SURREAL_URL=ws://surrealdb:8000/rpc
      - SURREAL_USER = ریشه
      - SURREAL_PASSWORD=رمز عبور
      - SURREAL_NAMESPACE=open_notebook
      - SURREAL_DATABASE=نوت بوک باز
    حجم ها:
      - ./notebook_data:/app/data
    بستگی_به:
      - surrealdb
    راه اندازی مجدد: همیشه

  اولاما:
    تصویر: olama/olama:جدیدترین
    پورت ها:
      - "11434:11434"
    حجم ها:
      - ./olama_models:/root/.ollama
    راه اندازی مجدد: همیشه
    # اختیاری: پشتیبانی از GPU را در صورت وجود تنظیم کنید
    #استقرار:
    # منابع:
    # رزرو:
    # دستگاه:
    # - درایور: nvidia
    # تعداد: 1
    # قابلیت: [gpu]

```

**ویرایش فایل:**
- «رشته تغییر من به یک مخفی» را با راز خود جایگزین کنید (هر رشته ای کار می کند)

## مرحله 3: شروع خدمات (1 دقیقه)

ترمینال را در پوشه «open-notebook-local» خود باز کنید:

``باش
docker آهنگسازی -d
```

10-15 ثانیه صبر کنید تا همه سرویس ها شروع شوند.

## مرحله 4: دانلود یک مدل (2-3 دقیقه)اولاما حداقل به یک مدل زبان نیاز دارد. یکی را انتخاب کنید:

``باش
# سریعترین و کوچکترین (برای آزمایش توصیه می شود)
docker exec open-notebook-local-ollama-1 ollama pull mistral

# OR: کیفیت بهتر اما کندتر
docker exec open-notebook-local-ollama-1 ollama pull neuro-chat

# یا: حتی با کیفیت بهتر، VRAM بیشتری مورد نیاز است
docker exec open-notebook-local-ollama-1 ollama pull llama2
```

این مدل را دانلود می کند (بسته به اینترنت شما 1-5 دقیقه طول می کشد).

## مرحله 6: ارائه دهنده Ollama را پیکربندی کنید (1 دقیقه)

1. به **تنظیمات** → **کلیدهای API** بروید
2. روی **افزودن اعتبار** کلیک کنید
3. ارائه دهنده را انتخاب کنید: **Ollama**
4. نامی به آن بدهید (مثلاً "Ollama محلی")
5. URL پایه را وارد کنید: «http://ollama:11434».
6. روی **ذخیره** کلیک کنید
7. روی **Test Connection** کلیک کنید - باید موفقیت را نشان دهد
8. روی **Discover Models** → **Register Models** کلیک کنید

---

## مرحله 7: پیکربندی مدل محلی (1 دقیقه)

1. به **تنظیمات** → **مدل** بروید
2. مجموعه:
   - **مدل زبان**: `ollama/mistral` (یا هر مدلی که دانلود کردید)
   - **مدل جاسازی**: `ollama/nomic-embed-text` (در صورت عدم وجود، دانلود خودکار)
3. روی **ذخیره** کلیک کنید

---

## مرحله 8: اولین نوت بوک خود را بسازید (1 دقیقه)

1. روی **New Notebook** کلیک کنید
2. نام: "تحقیق خصوصی من"
3. روی **ایجاد** کلیک کنید

## مرحله 9: اضافه کردن محتوای محلی (1 دقیقه)

1. روی **افزودن منبع** کلیک کنید
2. **متن** را انتخاب کنید
3. مقداری متن یا یک سند محلی را جایگذاری کنید
4. روی **افزودن** کلیک کنید

## مرحله 10: با محتوای خود گپ بزنید (1 دقیقه)

## عیب یابی

### "olama: دستور پیدا نشد"

نام تصویر داکر ممکن است متفاوت باشد:
``باش
docker ps # نام ظرف اوللاما را پیدا کنید
docker exec <container_name> ollama pull mistral
```### مدل دانلود گیر کرده است

اتصال اینترنت را بررسی کنید و راه اندازی مجدد کنید:
``باش
docker compose راه اندازی مجدد olama
```

سپس دستور pull مدل را دوباره امتحان کنید.

### خطای "آدرس در حال استفاده است".

``باش
docker آهنگسازی
docker آهنگسازی -d
```

### عملکرد پایین

بررسی کنید که آیا GPU در دسترس است:
``باش
# GPU های موجود را نمایش دهید
docker exec open-notebook-local-ollama-1 ollama ps

# GPU را در docker-compose.yml فعال کنید
```

سپس راه اندازی مجدد: «docker compose restart olama».

### اضافه کردن مدل های بیشتر

``باش
# مدل های موجود را لیست کنید
docker exec open-notebook-local-ollama-1 ollama list

# مدل اضافی را بکشید
docker exec open-notebook-local-ollama-1 ollama pull neuro-chat
```

## جایگزین: استفاده از LM Studio به جای Ollama

** رابط کاربری گرافیکی را ترجیح می دهید؟** LM Studio برای کاربران غیر فنی راحت تر است:

1. دانلود LM Studio: https://lmstudio.ai
2. برنامه را باز کنید، یک مدل را از کتابخانه دانلود کنید
3. به تب "سرور محلی" بروید، سرور را راه اندازی کنید (پورت 1234)
4. در Open Notebook، به **تنظیمات** → **کلیدهای API** بروید.
5. روی **Add Credential** کلیک کنید → **OpenAI-Compatible** را انتخاب کنید
6. URL پایه را وارد کنید: «http://host.docker.internal:1234/v1».
۷. کلید API را وارد کنید: «lm-studio» (جای‌بان)
8. روی **ذخیره** و سپس روی **Test Connection** کلیک کنید.
9. در تنظیمات → مدل ها → مدل LM Studio خود را انتخاب کنید

**توجه**: LM Studio خارج از داکر اجرا می شود، برای اتصال از «host.docker.internal» استفاده کنید.

## جلوتر رفتن

- **تغییر مدل**: در تنظیمات → مدل ها در هر زمان تغییر دهید
- **اضافه کردن مدل های بیشتر**:
  - Ollama: «ollama pull <model>» را اجرا کنید، سپس مدل‌ها را دوباره از اعتبارنامه کشف کنید
  - LM Studio: از کتابخانه برنامه دانلود کنید
- ** استقرار در سرور **: همان docker-compose.yml در همه جا کار می کند
- **استفاده از ابر ترکیبی**: برخی از مدل های محلی را نگه دارید، اعتبار ارائه دهنده ابر را برای کارهای پیچیده اضافه کنید
