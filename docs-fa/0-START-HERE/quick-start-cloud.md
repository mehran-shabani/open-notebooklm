# شروع سریع - ارائه دهندگان هوش مصنوعی ابری (5 دقیقه)

Open Notebook را با **Anthropic، Google، Groq یا سایر ارائه دهندگان ابری** اجرا کنید. همان سادگی OpenAI، با انتخاب های بیشتر.

## پیش نیاز

1. **Docker Desktop** نصب شده است
   - [از اینجا دانلود کنید](https://www.docker.com/products/docker-desktop/)
   - از قبل دارید؟ به مرحله 2 بروید

2. **کلید API** از ارائه دهنده انتخابی شما:
   - **OpenRouter** (100+ مدل، یک کلید): https://openrouter.ai/keys
   - **آنتروپیک (کلود)**: https://console.anthropic.com/
   - **گوگل (جمینی)**: https://aistudio.google.com/
   - **Groq** (سطح سریع و رایگان): https://console.groq.com/
   - **Mistral**: https://console.mistral.ai/
   - **DeepSeek**: https://platform.deepseek.com/
   - **xAI (Grok)**: https://console.x.ai/

## مرحله 1: ایجاد پیکربندی (1 دقیقه)

یک پوشه جدید «open-notebook» ایجاد کنید و این فایل را اضافه کنید:

**docker-compose.yml**:
`` یامل
خدمات:
  surrealdb:
    تصویر: surrealdb/surrealdb:v2
    دستور: start --user root --pass password --bind 0.0.0.0:8000 rocksdb:/mydata/mydatabase.db
    پورت ها:
      - "8000:8000"
    حجم ها:
      - ./سوررئال_داده:/mydata
    # بررسی سلامت حذف شد زیرا تصویر v2 برای اجرای wget/curl بسیار کم است
    راه اندازی مجدد: همیشه

  open_notebook:
    تصویر: lfnovo/open_notebook:v1-latest
    pull_policy: همیشه
    پورت ها:
      - "8502:8502" # رابط کاربری وب
      - "5055:5055" # API
    محیط زیست:
      - OPEN_NOTEBOOK_ENCRYPTION_KEY=تغییر-me-to-a-secret-string
      - SURREAL_URL=ws://surrealdb:8000/rpc
      - SURREAL_USER = ریشه
      - SURREAL_PASSWORD=رمز عبور
      - SURREAL_NAMESPACE=open_notebook
      - SURREAL_DATABASE=نوت بوک باز
    حجم ها:
      - ./notebook_data:/app/data
    بستگی_به:
      - surrealdb
    راه اندازی مجدد: همیشه

```

**ویرایش فایل:**
- «رشته تغییر من به یک مخفی» را با راز خود جایگزین کنید (هر رشته ای کار می کند)

مرورگر خود را باز کنید:

http://localhost:8502
```

## مرحله 4: ارائه دهنده هوش مصنوعی خود را پیکربندی کنید (1 دقیقه)

1. به **تنظیمات** → **کلیدهای API** بروید
2. روی **افزودن اعتبار** کلیک کنید
3. ارائه دهنده خود را انتخاب کنید (به عنوان مثال، Anthropic، Google، Groq، OpenRouter)
4. نامی به آن بدهید، کلید API خود را بچسبانید
5. روی **ذخیره** کلیک کنید
6. روی **Test Connection** کلیک کنید - باید موفقیت را نشان دهد7. روی **Discover Models** → **Register Models** کلیک کنید

مدل های ارائه دهنده شما اکنون در دسترس هستند!

> ** ارائه دهنده های متعدد **: می توانید اعتبارنامه ها را برای هر تعداد ارائه دهنده که می خواهید اضافه کنید. فقط این مرحله را برای هر ارائه دهنده تکرار کنید.

## مرحله 5: مدل خود را پیکربندی کنید (1 دقیقه)

1. به **تنظیمات** (نماد چرخ دنده) بروید
2. به **مدل ها** بروید
3. مدل ارائه دهنده خود را انتخاب کنید:

| ارائه دهنده | مدل پیشنهادی | یادداشت ها |
|----------|------------------|-------|
| **OpenRouter** | "انتروپیک/کلود-3.5-غزل" | دسترسی به 100+ مدل |
| **آنتروپیک** | `کلود-3-5-آخرین غزل` | بهترین استدلال |
| **گوگل** | `gemini-2.0-flash` | زمینه بزرگ، سریع |
| **گروک** | `lama-3.3-70b- همه کاره` | فوق سریع |
| **میسترال** | `mistral-large-جدیدترین` | گزینه اروپایی قوی |

4. روی **ذخیره** کلیک کنید

## عیب یابی

### خطای "مدل پیدا نشد".

1. به **تنظیمات** → **کلیدهای API** بروید
2. روی **Test Connection** روی اعتبار خود کلیک کنید
3. اگر معتبر است، روی **Discover Models** → **Register Models** کلیک کنید
4. بررسی کنید که اعتبار/دسترسی برای مدل دارید

### "نمی توان به سرور متصل شد"

``باش
docker ps # همه سرویس‌های در حال اجرا را بررسی کنید
docker compose logs # مشاهده گزارش‌ها
docker compose راه اندازی مجدد # همه چیز را راه اندازی مجدد کنید
```

### مسائل خاص ارائه دهنده

**Anthropic**: مطمئن شوید که کلید با "sk-ant-" شروع می شود
**Google**: از کلید AI Studio استفاده کنید، نه از Cloud Console
**Groq**: ردیف رایگان دارای محدودیت نرخ است. در صورت نیاز ارتقا دهید
