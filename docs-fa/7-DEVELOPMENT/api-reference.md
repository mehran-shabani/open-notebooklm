# مرجع API

REST API را برای Open Notebook کامل کنید. تمام نقاط پایانی از باطن API (پیش‌فرض: «http://localhost:5055») ارائه می‌شوند.

** URL پایه **: `http://localhost:5055` (توسعه) یا URL تولید خاص محیط

**اسناد تعاملی**: از رابط کاربری داخلی Swagger FastAPI در «http://localhost:5055/docs» برای آزمایش و کاوش زنده استفاده کنید. این مرجع اولیه برای تمام نقاط پایانی، طرح‌واره‌های درخواست/پاسخ و آزمایش بلادرنگ است.

---

## شروع سریع

### 1. احراز هویت

مبتنی بر رمز عبور ساده (فقط برای توسعه):

``باش
حلقه http://localhost:5055/api/notebooks \
  -H "Authorization: Bearer your_password"
```

**⚠️ تولید **: با OAuth/JWT جایگزین کنید. برای جزئیات بیشتر به [پیکربندی امنیتی] (../5-CONFIGURATION/security.md) مراجعه کنید.

### 2. Base API Flow

اکثر عملیات ها از این الگو پیروی می کنند:
1. یک **نوت بوک** (ظرفی برای تحقیق) ایجاد کنید
2. **منابع** را اضافه کنید (PDF، URL، متن)
3. پرس و جو از طریق **چت** یا **جستجو**
4. مشاهده نتایج و **یادداشت**

### 3. تست نقاط پایانی

به جای به خاطر سپردن نقاط پایانی، از اسناد تعاملی API استفاده کنید:
- به «http://localhost:5055/docs» بروید
- درخواست ها را مستقیماً در مرورگر امتحان کنید
- طرح های درخواست/پاسخ را در زمان واقعی مشاهده کنید
- با داده های خود تست کنید

## مروری بر نقاط پایانی API

### انواع منابع اصلی

**نوت بوک** - پروژه های تحقیقاتی حاوی منابع و یادداشت ها
- "GET/POST / notebooks" - فهرست و ایجاد کنید
- `GET/PUT/DELETE /notebooks/{id}` - خواندن، به‌روزرسانی، حذف

**منابع** - موارد محتوا (PDF، URL، متن)
- "GET/POST /sources" - فهرست و اضافه کردن محتوا
- "GET /sources/{id}" - جزئیات منبع را واکشی کنید
- "POST /sources/{id}/retry" - پردازش ناموفق را دوباره امتحان کنید
- "GET /sources/{id}/download" - فایل اصلی را دانلود کنید

** یادداشت ها ** - یادداشت های تحقیقاتی ایجاد شده توسط کاربر یا هوش مصنوعی
- "GET/POST / notes" - فهرست و ایجاد کنید
- «GET/PUT/DELETE /notes/{id}» - خواندن، به‌روزرسانی، حذف

** چت ** - رابط هوش مصنوعی مکالمه
- "GET/POST /chat/sessions" - جلسات چت را مدیریت کنید
- "POST /chat/execute" - پیام ارسال کنید و پاسخ دریافت کنید
- "POST /chat/context/build" - زمینه را برای چت آماده کنید

**جستجو** - محتوا را بر اساس متن یا شباهت معنایی پیدا کنید
- `POST / Search` - جستجوی متن کامل یا برداری
- "POST / Ask" - یک سوال بپرسید (جستجو + ترکیب)

**تحولات** - درخواست های سفارشی برای استخراج بینش
- "GET/POST /transformations" - قوانین استخراج سفارشی ایجاد کنید
- "POST /sources/{id}/insights" - تغییر شکل را در منبع اعمال کنید

**مدل ها** - ارائه دهندگان هوش مصنوعی را پیکربندی کنید
- "GET / Models" - مدل های موجود
- "GET /models/defaults" - پیش‌فرض‌های فعلی- "POST /models/config" - پیش فرض ها را تنظیم کنید

** اعتبار ** - اعتبار ارائه دهنده هوش مصنوعی را مدیریت کنید
- "GET/POST /Credentials" - اعتبارنامه ها را فهرست کرده و ایجاد کنید
- "GET/PUT/DELETE /credentials/{id}" - عملیات CRUD
- `POST /credentials/{id}/test` - اتصال را آزمایش کنید
- "POST /credentials/{id}/discover" - مدل‌های ارائه‌دهنده را کشف کنید
- `POST /credentials/{id}/register-models` - ثبت مدل‌های کشف‌شده
- "GET /Credentials/status" - نمای کلی وضعیت ارائه دهنده
- "GET /Credentials/env-status" - وضعیت متغیر محیطی
- `POST /credentials/migrate-from-env` - انتقال env vars به اعتبارنامه

**سلامت و وضعیت**
- "دریافت / سلامت" - بررسی سلامت
- "GET /commands/{id}" - عملیات ناهمگام را ردیابی کنید

## احراز هویت

### فعلی (توسعه)

همه درخواست‌ها نیاز به هدر رمز عبور دارند:

``باش
curl -H "Authorization: Bearer your_password" http://localhost:5055/api/notebooks
```

رمز عبور از طریق متغیر محیطی «OPEN_NOTEBOOK_PASSWORD» پیکربندی شد.

>**

### تولید

**⚠️ امن نیست. ** جایگزین کنید با:
- OAuth 2.0 (توصیه می شود)
- توکن های JWT
- کلیدهای API

برای تنظیم تولید به [پیکربندی امنیتی] (../5-CONFIGURATION/security.md) مراجعه کنید.

## الگوهای رایج

### صفحه بندی

``باش
# فهرست منابع با محدودیت / افست
حلقه "http://localhost:5055/sources?limit=20&offset=10"
```

### فیلتر کردن و مرتب سازی

``باش
# فیلتر بر اساس نوت بوک، مرتب سازی بر اساس تاریخ
حلقه "http://localhost:5055/sources?notebook_id=notebook:abc&sort_by=created&sort_order=asc"
```

### عملیات Async

برخی از عملیات (پردازش منبع، تولید پادکست) بلافاصله با یک شناسه فرمان باز می گردند:

``باش
# عملیات ناهمگام را ارسال کنید
curl -X POST http://localhost:5055/sources -F async_processing=true
# پاسخ: {"id": "source:src001"، "command_id": "command:cmd123"}

# وضعیت نظرسنجی
حلقه http://localhost:5055/commands/command:cmd123
```

### پاسخ های جریانی

نقطه پایانی `/ask` پاسخ ها را به عنوان رویدادهای ارسال شده از سرور پخش می کند:

``باش
curl -N 'http://localhost:5055/ask' \
  -H "Content-Type: application/json" \
  -d '{"question": "AI چیست؟"}'

# خروجی: داده: {"type":"strategy",...}
# داده: {"type":"answer"،...}
# داده: {"type":"final_answer"،...}
```

### آپلود فایل چند قسمتی

``باش
curl -X POST http://localhost:5055/sources \
  -F "type=upload" \
  -F "notebook_id=notebook:abc" \
  -F "file=@document.pdf"
```

## رسیدگی به خطاهمه خطاها JSON را با کد وضعیت برمی گرداند:

``json
{"detail": "Notebook یافت نشد"}
```

### کدهای وضعیت رایج

| کد | معنی | مثال |
|------|---------|---------|
| 200 | موفقیت | عملیات تکمیل شد |
| 400 | درخواست بد | ورودی نامعتبر |
| 404 | یافت نشد | منبع وجود ندارد |
| 409 | درگیری | منبع از قبل وجود دارد |
| 500 | خطای سرور | خطای پایگاه داده/پردازش |

## نکاتی برای توسعه دهندگان

1. ** با اسناد تعاملی شروع کنید ** (`http://localhost:5055/docs`) - این مرجع قطعی است
2. **فعال کردن ورود به سیستم** برای اشکال زدایی (لاگ های API را بررسی کنید: "گزارش های docker")
3. **نقاط پایانی پخش جریانی** به مدیریت خاصی نیاز دارد (رویدادهای ارسال شده از سرور، نه JSON استاندارد)
4. **عملیات Async** بلافاصله برمی گردند. همیشه قبل از فرض تکمیل وضعیت نظرسنجی را انجام دهید
5. **جستجوی برداری** به پیکربندی مدل جاسازی نیاز دارد («/models» را بررسی کنید)
6. **نسخ مدل** به ازای هر درخواست است. تنظیم در بدنه، پیکربندی نیست
7. **CORS فعال ** در حال توسعه. پیکربندی برای تولید

---

## مسیر یادگیری

1. **Authentication**: هدر "X-Password" را به همه درخواست ها اضافه کنید
2. **یک دفترچه یادداشت بسازید**: `POST /notebooks` با نام و توضیحات
3. **یک منبع** اضافه کنید: «POST /sources» با فایل، URL یا متن
4. **از محتوای خود پرس و جو کنید**: برای پرسیدن سوال، 'POST /chat/execute'
5. **کاوش ویژگی های پیشرفته **: جستجو، تحولات، جریان

---

## ملاحظات تولید

- تأیید رمز عبور را با OAuth/JWT جایگزین کنید (به [امنیت] (../5-CONFIGURATION/security.md) مراجعه کنید)
- اضافه کردن محدودیت نرخ از طریق پراکسی معکوس (Nginx، CloudFlare، Kong)
- محدودیت های CORS را فعال کنید (در حال حاضر به همه مبدا اجازه می دهد)
- از HTTPS از طریق پراکسی معکوس استفاده کنید (به [Reverse Proxy] (../5-CONFIGURATION/reverse-proxy.md) مراجعه کنید)
- راه‌اندازی استراتژی نسخه‌سازی API (در حال حاضر ضمنی)

برای راه اندازی کامل تولید، به [پیکربندی امنیتی](../5-CONFIGURATION/security.md) و [تنظیم پروکسی معکوس] (../5-CONFIGURATION/reverse-proxy.md) مراجعه کنید.
