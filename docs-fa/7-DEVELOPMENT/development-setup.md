# راه اندازی توسعه محلی

این راهنما شما را در راه‌اندازی Open Notebook برای توسعه محلی راهنمایی می‌کند. این مراحل را دنبال کنید تا پشته کامل روی دستگاه شما اجرا شود.

## پیش نیاز

قبل از شروع، مطمئن شوید که موارد زیر را نصب کرده اید:

- ** Python 3.11+** - با: "python --version" بررسی کنید
- **uv** (توصیه می شود) یا **pip** - نصب از: https://github.com/astral-sh/uv
- ** SurrealDB ** - از طریق داکر یا باینری (به زیر مراجعه کنید)
- ** داکر ** (اختیاری) - برای پایگاه داده کانتینری
- **Node.js 18+** (اختیاری) - برای توسعه frontend
- **Git** - برای کنترل نسخه

## مرحله 3: متغیرهای محیطی

یک فایل «.env» در ریشه پروژه با پیکربندی خود ایجاد کنید:

``باش
# از نمونه کپی کنید
cp .env.example .env
```

«.env» را با تنظیمات خود ویرایش کنید:

``باش
# پایگاه داده
SURREAL_URL=ws://localhost:8000/rpc
SURREAL_USER=ریشه
SURREAL_PASSWORD=رمز عبور
SURREAL_NAMESPACE=open_notebook
SURREAL_DATABASE=توسعه

# رمزگذاری اعتبار (برای ذخیره کلیدهای API لازم است)
OPEN_NOTEBOOK_ENCRYPTION_KEY=my-dev-secret-key

# اپلیکیشن
APP_PASSWORD= # حفاظت از رمز عبور اختیاری
DEBUG=true
LOG_LEVEL=DEBUG
```

### پیکربندی ارائه دهنده هوش مصنوعی

پس از راه‌اندازی API و frontend، ارائه‌دهنده هوش مصنوعی خود را از طریق تنظیمات UI پیکربندی کنید:

1. **http://localhost:3000** → **تنظیمات** → **کلیدهای API** را باز کنید
2. روی **افزودن اعتبار** → ارائه دهنده خود را انتخاب کنید
3. کلید API خود را وارد کنید (از داشبورد ارائه دهنده دریافت کنید)
4. روی **ذخیره** و سپس روی **Test Connection** کلیک کنید.
5. روی **Discover Models** → **Register Models** کلیک کنید

ارائه دهندگان محبوب:
- **OpenAI** - https://platform.openai.com/api-keys
- **آنتروپیک (کلود)** - https://console.anthropic.com/
- **Google** - https://ai.google.dev/
- **Groq** - https://console.groq.com/

برای توسعه محلی، می توانید از:
- **Ollama** - به صورت محلی بدون کلیدهای API اجرا شود (به "Ollama محلی" در زیر مراجعه کنید)

> **توجه:** متغیرهای محیطی کلید API (به عنوان مثال، «OPENAI_API_KEY») منسوخ شده‌اند. به جای آن از تنظیمات رابط کاربری برای مدیریت اعتبارنامه ها استفاده کنید.

## مرحله 4: SurrealDB را شروع کنید

### گزینه A: استفاده از Docker (توصیه می شود)

``باش
# SurrealDB را در حافظه راه اندازی کنید
docker run -d --name surrealdb -p 8000:8000 \surrealdb/surrealdb:v2 start \
  -- کاربر ریشه -- رمز عبور \
  --bind 0.0.0.0:8000 حافظه

# یا با ذخیره سازی مداوم
docker run -d --name surrealdb -p 8000:8000 \
  -v surrealdb_data:/data \
  surrealdb/surrealdb:v2 start \
  -- کاربر ریشه -- رمز عبور \
  --bind 0.0.0.0:8000 فایل:/data/surreal.db
```

## مرحله 5: مهاجرت پایگاه داده را اجرا کنید

هنگامی که API را راه اندازی می کنید، انتقال پایگاه داده به طور خودکار اجرا می شود. اولین راه‌اندازی هرگونه مهاجرت معلق را اعمال خواهد کرد.

برای تأیید انتقال به صورت دستی:

``باش
# API مهاجرت ها را هنگام راه اندازی اجرا می کند
uv run python -m api.main
```

گزارش‌ها را بررسی کنید - باید پیام‌هایی مانند:
```
اجرای مهاجرت 001_initial_schema
اجرای مهاجرت 002_add_vectors
...
مهاجرت ها با موفقیت انجام شد
```

## مرحله 6: سرور API را راه اندازی کنید

```

## عیب یابی

### "اتصال رد شد" در SurrealDB

**مشکل**: API نمی تواند به SurrealDB متصل شود

**راه حل**:
1. بررسی کنید که آیا SurrealDB در حال اجرا است: `docker ps | grep surrealdb`
2. URL را در «.env» تأیید کنید: باید «ws://localhost:8000/rpc» باشد
3. SurrealDB را مجدداً راه اندازی کنید: «docker stop surrealdb && docker rm surrealdb».
4. سپس با: «docker run -d --name surrealdb -p 8000:8000 surrealdb/surrealdb:v2 start --user root --pass password --bind 0.0.0.0:8000 memory» راه اندازی مجدد کنید.

### "آدرس در حال استفاده است"

**مشکل**: پورت 5055 یا 3000 در حال حاضر در حال استفاده است

**راه حل**:
``باش
# فرآیند را با استفاده از پورت پیدا کنید
lsof -i :5055 # پورت 5055 را بررسی کنید

# فرآیند کشتن (macOS/Linux)
کشتن -9 <PID>

# یا از پورت های مختلف استفاده کنید
uvicorn api.main:app --port 5056
```

### خطاهای ماژول پیدا نشد

**مشکل**: خطاهای وارد کردن هنگام اجرای API

**راه حل**:
``باش
# وابستگی ها را دوباره نصب کنید
همگام سازی uv

# یا با پیپ
pip install -e .
```

### خرابی انتقال پایگاه داده

**مشکل**: API با خطاهای مهاجرت شروع نمی شود

**راه حل**:
1. بررسی کنید که SurrealDB در حال اجرا است: «curl http://localhost:8000/»
2. اعتبارنامه ".env" را بررسی کنید که با تنظیمات SurrealDB شما مطابقت دارد
3. گزارش‌ها را برای خطای خاص مهاجرت بررسی کنید: `make api 2>&1 | مهاجرت grep -i4. بررسی وجود پایگاه داده: کنسول SurrealDB را در http://localhost:8000/ بررسی کنید

### مهاجرت ها اعمال نمی شوند

**مشکل**: طرح واره پایگاه داده قدیمی به نظر می رسد

**راه حل**:
1. راه اندازی مجدد API - مهاجرت ها در هنگام راه اندازی اجرا می شوند: "make api".
2. بررسی گزارش‌ها نشان می‌دهد «مهاجرت‌ها با موفقیت انجام شد»
3. بررسی کنید پوشه `/migrations/` وجود داشته باشد و دارای فایل باشد
4. بررسی کنید که SurrealDB قابل نوشتن است و در حالت فقط خواندنی نیست
